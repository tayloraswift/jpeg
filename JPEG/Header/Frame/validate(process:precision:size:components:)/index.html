Static Method

validate(process:precision:size:components:)

Creates a frame header after validating the given field values.

Declaration

static func validate(process:JPEG.Process, precision:Int, size:(x:Int, y:Int), components:[JPEG.Component.Key: JPEG.Component]) throws -> Self

Parameters

process

The coding process used by the image.

precision

The bit depth of the image. If the process is baseline, this parameter must be 8. If the process is extended(coding:differential:) or progressive(coding:differential:), this parameter must be either 8 or 12. If the process is lossless(coding:differential:), this parameter must be within the interval 2 ... 16.

size

The size of the image, in pixels. Passing a negative height will result in a precondition failure. Passing a negative or zero width will result in a ParsingError. This constructor treats the two failure conditions differently because the latter one is the only one that can occur when parsing a frame header from input data.

components

The components in the image. This dictionary must have at least one element. If the process is progressive(coding:differential:), it can have no more than four elements. The sampling factors of each component must be within the interval 1 ... 4 in both directions. if the process is baseline, the components can only use the quantization table selectors \.0 and \.1.

Return value

A frame header.

Overview

If the given parameters are not consistent with one another, and the JPEG standard, this function will throw a ParsingError, unless otherwise noted.
