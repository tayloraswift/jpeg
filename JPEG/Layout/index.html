Generic Structure

JPEG.Layout

A specification of the components, coding process, table assignments, and scan progression of an image.

Declaration

struct Layout<Format> where Format:JPEG.Format

Overview

This structure records both the recognized components and the resident components in an image. We draw this distinction because the planes property is allowed to include definitions for components that are not part of format.components. Such components will not recieve a plane in the Data types, but will be ignored by the scan decoder without errors.

Non-recognized components can only occur in images decoded from JPEG files, and only when using a custom Format type, as the built-in JPEG.Common color format will never accept any component declaration in a frame header that it does not also recognize. When encoding images to JPEG files, all declared resident components must also be recognized components.

Image modes

The color format of the image.

Component membership

let residents:[JPEG.Component.Key: Int]

The set of color components declared (or to-be-declared) in the image frame header.

var recognized:[JPEG.Component.Key]

The set of color components in the color format of this image. This set is always a subset of the resident components in the image.

Image structure

var planes:[(component:JPEG.Component, qi:JPEG.Table.Quantization.Key)]

The descriptor array for the planes in the image.

var definitions:[(quanta:[JPEG.Table.Quantization.Key], scans:[JPEG.Scan])]

The sequence of scan and table definitions in the image file.

Instance properties

var components:[JPEG.Component.Key: (factor:(x:Int, y:Int), qi:JPEG.Table.Quantization.Key)]

A dictionary mapping all of the resident components in the image to their sampling factors and quanta keys.

var scale:(x:Int, y:Int)

The size of the minimum-coded unit of the image, in data units.

var scans:[JPEG.Header.Scan]

The scan decomposition of this image layout, filtered to include only recognized components.
